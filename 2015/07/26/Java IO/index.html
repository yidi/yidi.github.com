Java IO分类

Java中大部分涉及流IO的都是在java.io包内。而大部分涉及文件IO的都在java.nio.file包内。
很多人将JDK1.4之前的Java IO称为所谓的BIO，及Blocking IO，而将JDK1.4之后引入的新的IO称为No-blocking IO，这些说法都没有完全的根据，但是确实在JDK1.4以后引入了新的java.nio包提供了支持非阻塞IO的方式，而之前的IO都是基于阻塞IO的。另外在JDK1.7后又产生了所谓AIO，即 Asynchronous
IO，人们把之前的IO归类为Synchronous IO，这里我们不过多讨论这些IO 的分类。在特定知识点处再说明。关于IO在Java的历次版本升级中的情况请参考：官方文档（Enhancements in Java I/O）

IO Stream

Byte Streams

所有Byte Streams都继承自InputStream 和OutputStream，就是以字节为单位进行读写。需要注意的是使用完IO流后要进行close()操作。

Character Streams

所有Character Streams都继承自Reader和Writer，Java平台用Unicode编码，而Character Streams会自动处理本地字符集与Unicode之间的转换。 这在国际化的时候非常实用，即使不考虑国际化，使用Character Streams也不用担心字符集的问题了。Character Streams通常应用于包括很多行的场景，而不只是一个简单的字符串，这种场景下就需要处理换行符或者对行单独处理. BufferedReader和PrintWriter对应的提供了readLine()和println()方法，会根据操作系统的不同进行处理。

Buffered Streams

不论是Byte Streams还是Character Streams都是非缓冲流，这样每次进行IO请求的时候都要进行磁盘访问或者网络传输等昂贵的操作，而Buffered Streams可以减少这些开销，只要将非缓冲流对象传入缓存流类型构造器（也即是流装饰，装饰者模式应用的范例）即可将一个非缓冲流转换成一个缓冲流。Byte Streams对应的是 BufferedInputStream 和 BufferedOutputStream 而Character Streams对应的是BufferedReader和BufferedWriter。示例如下：

inputStream = new BufferedReader(new FileReader("xanadu.txt"));
outputStream = new BufferedWriter(new FileWriter("characteroutput.txt"));

Scanning

Scanner类将输入分割为一个个的单词（token）默认的分隔符为空白字符（Character.isWhitespace）而Formatting 则将数据按照要求进行格式化。
Scanner可以通过useDelimiter()函数设定分隔符，useLocale()处理一些区域差异化。

Formatting

实现格式化的类是字符流类PrintWriter或者字节流类PrintStream,关于格式化用的并不多，如果需要可参考对应的函数的相关说明即可。

IO Command Line

命令行下的IO有标准流：标准输入System.in,标准输出：System.out，及标准错误System.err.这些不多介绍，另外还有Console类型，这个类型比以上三个类型更高级。尤其是处理密码安全输入等场景。输入的密码在用户屏幕是不可见的。

Data Stream

数据流支持所有的基本数据类型包括字符串，对应的类型为DataInputStream和DataOutputStream，所有输入数据流都实现了DataInput接口而所有输出数据流都实现了DataOutput接口。数据流写出用writeDouble()、writeInt()、.writeUTF()等函数，读入用readDouble()、readInt()、readUTF()等函数，并且通过捕获EOFException异常来探测数据的结束。输入数据流不会检验读入的数据流中的数据值是否正确以及数据所在位置，这完全需要程序员去控制。
需要说明是使用浮点数表示精确的货币值是很糟糕的，因为浮点数并不精确，尤其是处理“0.1”这样的小数部分。如果需要处理货币值应该使用BigDecimal，但是BigDecimal是一个对象类型，而不是数据类型，因此需要使用Object Stream处理。

Object Stream

Object Stream支持对象的IO。对象流对应的类型为 ObjectInputStream和ObjectOutputStream，他们分别实现了 ObjectInput和ObjectOutput接口，这两个接口又分别是DataInput和DataOutput的子接口，也即Object Stream实现了Data Stream的所有方法。

writeObject()和readObject()的使用方法和数据流中的方式使用方式一样，但是这里包含一些复杂的对象管理逻辑。一些对象中会包含对另一些对象的引用。readObject()要向从对象流中重构对象，就必须同时能重构对象所引用的其他对象，而这些额外被引用的对象也有可能还引用了另外的对象等等。这就要求writeObject()方法在将对象写入对象流的时候遍历对象的引用网并将所有涉及到的对象都写入流中。也即对一个对象进行writeObject()操作，可能会导致写入多个对象。

例如对象a包含对对象b和对象c的引用，而对象b又包含对对象d和e的引用，调用writeobject(a)就会不仅仅写入a，同时也会将b、c、d、e也写入。这时再调用readObject()的时候，这四个对象也就会被读回而构建起对象a。

但是考虑另外一种情况，就是两个对象a和b同时引用另一个对象c，如果对a、b两个对象进行readobject()操作他们引用的仍然是同一个对象c。一个流对一个对象只能有一份拷贝，即使这个拷贝被多个对象引用。也就是如果你对同一个对象进行两次调用writeobject()操作。对象只会写入一次，而会写入两次对这个对象的引用。（这也正是符合对象被引用的逻辑）例如：

Object ob = new Object();
out.writeObject(ob);
out.writeObject(ob);

Object ob1 = in.readObject();
Object ob2 = in.readObject();

这里读回的ob1和ob2是同一个对象。
但是如果将一个对象分别写入到两个不同的流中，则从这两个流中再读回的对象就会是两个不同的对象了。
